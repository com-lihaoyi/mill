==== Incremental Annotation Processing

Mill supports incremental annotation processing for processors that publish
Gradle-compatible metadata in `META-INF/gradle/incremental.annotation.processors`.
Supported kinds are `isolating`, `aggregating`, and `dynamic`.
Gradle documents what these categories mean and when each should be used in
https://docs.gradle.org/current/userguide/java_plugin.html#sec:incremental_annotation_processing[Incremental annotation processing].

Example metadata file:

.META-INF/gradle/incremental.annotation.processors
[source,text]
----
org.mapstruct.ap.MappingProcessor,isolating
com.google.auto.service.processor.AutoServiceProcessor,aggregating
----

When the active processors provide this metadata, Mill keeps Zinc incremental
compilation enabled and cleans up stale files generated by annotation
processors under `compile.dest/classes` when the corresponding sources are
removed or annotation processing is no longer active.

If any active processor does not provide supported incremental metadata, the
processor still works, but Mill falls back to non-incremental compilation for
that module to avoid leaving stale generated outputs behind.
If an upstream processor jar does not ship this metadata, you can add it in your
own module's `compile-resources/META-INF/gradle/incremental.annotation.processors`
to provide metadata for that processor yourself.
